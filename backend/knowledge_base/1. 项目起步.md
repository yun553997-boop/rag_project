## 1 拉取后端项目

- **说明**：拉取一个现成的开源后端项目（跑通真实接口） + 关键难点使用 Mock 补充
- **目标**：能把数据“展示”出来，并且能模拟“增删改查”的交互流程

一般步骤：

1. 在GitHub中搜索关键字，查找合适项目（主要查看readme文档、requirements.txt）：[医疗保健平台](https://github.com/devalentineomonya/health-care-management-system-python-fastapi)

2. 查看requirements.txt，配置环境、安装依赖、配置数据库，启动项目

   通常是

   ```
   uvicorn main:app --reload
   ```

   访问http://127.0.0.1:8000/docs，看到接口列表就算成功了！

3. 前端对接，编写api文件夹，使用axios写出对应的请求函数（后端返回的字段名如果是 `patient_name`，前端要用 `patientName`）

4. 编写登录逻辑，调用后端的 `/login` 接口，拿到 `access_token；`将 token 存入 `localStorage` 和 `Pinia`；在 axios 拦截器中，给每个请求头加上 `Authorization: Bearer <token>`。

5. 使用数据，渲染结构



## 2 拿到后端接口

###  第一步：环境准备

1. 下载**Docker Desktop**

   简介：

   - 后端项目被打包成了“容器镜像”，避免“在我机器上能跑”的问题

   - 使用Docker Desktop可以一键启动整个后端环境

     启动命令：

     ```powershell
     docker-compose up --build -d   #-d:后台运行
     ```

     > 启动之前，保证 **Docker Desktop**是启动状态（无需登录，以游客打开即可）

     查看当前容器状态：

     ```powershell
     docker compose ps
     ```

   - 不会污染本地操作系统，卸载时只需删除容器即可

     删除命令：

     ```powershell
     docker-compose down
     ```

2. 安装 **Git** (如果还没装)。

### 第二步：一键启动后端

1. 克隆项目

   ```bash
   git clone <你找到的这个项目的GitHub链接> healthcare-backend
   cd healthcare-backend
   ```

   *也可以在编译器中进行克隆*

2. 配置镜像加速器：

   1. 打开 **Docker Desktop** 界面。

   2. 点击右上角的 **齿轮图标 (Settings)**。

   3. 选择左侧的 **Docker Engine**。

   4. 在右侧的 JSON 配置中，填入可用的加速地址：

      ```json
      {
        "builder": {
          "gc": {
            "defaultKeepStorage": "20GB",
            "enabled": true
          }
        },
        "experimental": false,
        "registry-mirrors": [
          "https://docker.m.daocloud.io",
          "https://huecker.io",
          "https://dockerhub.timeweb.cloud",
          "https://noohub.ru"
        ]
      }
      ```

      *(注：以上地址为社区整理的可用源，时效性较强，若不可用请务必去阿里云注册账号获取专属加速链接)*

   5. 点击 **Apply & Restart**。Docker 会重启引擎。

3. 创建 `.env` 文件配置密码:

   PostgreSQL 的官方镜像在首次启动时，必须设置环境变量 POSTGRES_PASSWORD，设置密码

   在项目根目录下，创建一个名为 `.env` 的文件。

   ```env
   # 数据库密码 (必须设置，不能为空)
   DB_PASSWORD=zhaoyun20030504

   # RabbitMQ 账号密码
   RABBITMQ_USER=guest
   RABBITMQ_PASSWORD=guest

   # 后端密钥 (用于 JWT 签名等，随意设置一个长字符串)
   SECRET_KEY=my_super_secret_key_change_in_production
   ```

4. 启动

   ```powershell
   docker-compose up --build -d
   ```

5. 查看后端接口

   **等待几分钟**... 

   1. 查看端口号：

      查看 `docker-compose.yml` 中的 `ports` 部分，例如 `- "8080:8000"` 表示宿主机的 8080 端口映射到容器的 8000 端口。

   2. 在浏览器搜索：

      API 文档: <http://localhost:8000/docs>

      健康检查: <http://localhost:8000/health>

   3. 此时就可以看到Swagger UI界面（会列出了所有接口），如果没有显示就换浏览器 




## 3 测试接口

检查后端接口是否正常运行，**不要依赖 Swagger UI 的界面展示错误**

### 方法一：使用浏览器

条件：对于不需要登录认证、且是 `GET` 方法的简单接口，可以直接在浏览器测试。

步骤：

1. 基础 URL + 接口路径

   http://localhost:8000 + GET /api/patients = http://localhost:8000/api/patients

2. 访问并观察

   - 正常：页面显示 JSON 数据（如 `[{"id":1, "name":"..."}]`）或 XML/纯文本。
   - 需要认证：返回 `401 Unauthorized` 或 `403 Forbidden`。**这说明接口是活的**，只是没登录。
   - 服务挂了：浏览器显示 `无法连接`、`502 Bad Gateway` 或 `504 Gateway Time-out`。
   - 路径错误：返回 `404 Not Found`（说明该接口不存在或路径写错了）。



### 方法二：使用 Postman

功能最齐全，所以一般用它拿到token

步骤：

1. 打开 Postman，在左侧边栏点击 **+** 号，这会打开一个新的标签页，你会看到地址栏上方有一排下拉菜单（默认是 `GET`）。

2. 选择POST，填写接口地址（基础 URL + 接口路径）

   ```
   http://localhost:8000/api/auth/login
   ```

3. 点击 Body 标签 -> 选 raw -> 右边下拉选 JSON，输入参数（账号和密码）

   ```json
   {
     "username": "admin",
     "password": "123456"
   }
   ```

4. 点击send，登录成功（返回 200），响应返回token字段

> 方法一中使用get，通常要求登录认证，返回401。此时，我们拿到token就可以成功访问

步骤：

1. 新建Tab，方法选get，填写接口地址

   ```
   http://localhost:8000/api/patients
   ```

2. 点击 Authorization 标签 -> Type选择 Bearer Token -> 粘贴token字段

3. 点击send，返回200和数据

   ​

### 方法三：使用命令行 curl

直接使用，无需安装软件（但好像不咋行）

语法：

-  测试一个 GET 接口

  ```bash
  curl -v http://localhost:8080/api/patients  #-v:可以看到详细的请求头和响应头
  ```

- 如果接口需要 POST 数据

  ```bash
  curl -X POST http://localhost:8080/api/patients \
    -H "Content-Type: application/json" \
    -d '{"name": "Test User", "age": 25}'
  ```

解读结果：

| 现象                              | 含义                             | 结论                       |
| --------------------------------- | -------------------------------- | -------------------------- |
| 返回 JSON 数据                    | 业务逻辑正常，数据库连接正常     | 项目可用                   |
| 返回 401/403                      | 接口有保护机制，逻辑正常         | 项目可用，但先登录         |
| 返回 404                          | 路径不对，或该接口未实现         | 部分可用，需核对文档或源码 |
| 返回 500                          | 代码报错或数据库配置错误         | 有风险，需查看后端日志修复 |
| 浏览器/工具转圈后超时             | 服务未启动、防火墙拦截、端口错误 | 不可用，检查后端是否运行   |
| 直接拒绝连接 (Connection Refused) | 服务进程根本没跑起来             | 不可用，启动后端程序       |
