#### 2.1.4 token持久化

> 用户数据在Token（用来标识当前用户是否登录）中，而Token持续一段时间会过期。
>
> Pinia的储存是基于**内存（本地）**的，刷新就丢失，为保持登陆状态，需要持久化

**方法一：**

使用插件，官网：[开始 | Pinia Plugin Persistedstate](https://prazdevs.github.io/pinia-plugin-persistedstate/zh/guide/)

1. 安装

2. 在入口文件进行插件注册

3. 使用

   `stores/user.ts`

   ```js
   const useUserStore = defineStore('user', () => {
       ...
   },{
       persist: true,          //持久化
   })
   ```

**方法二：**

使用专门文件管理TOKEN

1. 写一个文件（utlis/token.ts），将其存储在本地

   ```ts
   // 大公司，会用一个token文件来管理token相关的逻辑
   export const SET_TOKEN = (token: string) => {
     localStorage.setItem("TOKEN", token);
   };

   export const GET_TOKEN = () => {
     return localStorage.getItem("TOKEN");
   };
   ```

2. 拿到并传入文件

   ```ts
   import { GET_TOKEN, SET_TOKEN } from "@/utils/token";
   import { ref } from 'vue';

   const useUserStore = defineStore('user',() => {
     const token = ref(GET_TOKEN() || "");
     //提交用户信息，进行登录
     const useLogin = async (data: loginFrom): Promise<UserState['token']> => {
       let res: loginResponseData = await reqLogin(data)
       //成功返回token,
       if (res.code == 200) {
         token.value = res.data.token as string
         SET_TOKEN(res.data.token as string);
         return 'ok'
         //失败返回登陆失败的信息
       } else {
         return Promise.reject(new Error(res.data.message))
       }
     }
     return {
       useLogin,
     }
   })

   export default useUserStore
   ```

### 2.2 优化

#### 2.2.1 定义类型

对useLogin的参数(data)、返回值(token)、登陆返回数据（res）进行定义类型

- 登录接口否会数据类型

  ```ts
  import type { loginFrom, loginResponseData } from '@/api/user/type'
  import type { UserState } from './type'

  const useUserStore = defineStore('user',() => {
      //提交用户信息，进行登录
      const useLogin = async (data: loginFrom): Promise<string|null> => {
        let res: loginResponseData = await reqLogin(data)
        //成功返回token
        if (res.code == 200) {
          const token = res.data.token as string     //断言
          return token 
        //失败返回登陆失败的信息
        } else {
          return Promise.reject(new Error(res.data.message))
        }
      }
      return {
        useLogin,
      }
    },
    //持久化
  )

  export default useUserStore
  ```

  > **断言**：作用是**手动指定一个值的类型**，告诉 TypeScript 编译器：“我确定这个值的类型是 `string`，请按照这个类型来处理它，不要进行类型检查报错”。

### 2.3 表单验证

> 要求：
>
> 1. 用户名：不能为空，字段长度在3~6
> 2. 密码：不能为空，且为6-12个字符

#### 2.3.1  初步验证

1. 绑定相应标签

   > el-from:
   >
   > 1. 绑定数据（:model="loginFrom"）
   > 2. 表单验证规则（:rules="rules"）
   >
   > el-form-item:
   >
   > 1.  model 的键名（prop="username"）**必填**
   >
   > el-input:
   >
   > 1. v-model="loginFrom.username"

   ```vue
   <el-form class="login_from" :model="loginFrom" :rules="rules">
     <el-form-item prop="username">
       <el-input v-model="loginFrom.username"></el-input>
     </el-form-item>
     ...
   </el-form>
   ```


2. 编写验证规则

   > 规则：
   >
   > 1. required：必填（选）项
   > 2. message：错误提示信息
   > 3. trigger：触发时机（blur：失去光标；change：文本发生变化）

   ```vue
   <script setup>
   const rules = {
     username: [
       { required: true, message: '账号不能为空', trigger: 'change' },
       { min: 3, max: 6, message: '账号长度至少3位', trigger: 'change' },
     ],
     password: [
       { required: true, message: '密码不能为空', trigger: 'change' },
       { min: 6, max: 12, message: '密码长度要求6~12位', trigger: 'change' },
     ]
   </script>
   ```
