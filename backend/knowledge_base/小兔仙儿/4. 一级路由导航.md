2. 封装接口

   `apis/category.ts`

   ```ts
   import httpInstance from '@/utils/http'

   export  function getTopCategoryAPI (id) {
     return httpInstance({
       url:'/category',         
       params:{                 //使用params方法传递参数（路由传参）
         id
       }
     })
   }
   ```


3. 渲染

   ```vue
   <script setup>
   import { getTopCategoryAPI } from '@/apis/category'
   import { ref, onMounted } from 'vue';
   import { useRoute } from 'vue-router';

   const categoryData = ref({})
   const route = useRoute()
   const getCategory = async() => {
       const res = await getTopCategoryAPI(route.params.id)     //接受路由参数
       categoryData.value = res.result
   }
   onMounted(() => getCategory())
   </script>

   <template>
      <el-breadcrumb-item :to="{ path: '/' }">首页</el-breadcrumb-item>
      <el-breadcrumb-item>{{ categoryData.name }}</el-breadcrumb-item>   //使用参数
   </template>
   ```

## 5 解决路由缓存问题**

路由缓存问题：

​	因为我们渲染所有一级导航内容都是使用生命周期钩子函数进行触发，就会导致我们在运行完一级 	  

​	导航后就不会再触发运行第二遍，**所以我们在点击首页以外的其他一级导航，不会实现跳转**。

解决办法：

 1. 让组件实例不在复用，强制销毁重建

    给`router-view`添加key

    ```vue
    <template>
        <div class="layout-index">
            <layout-eixed></layout-eixed>
            <layout-nav></layout-nav>
            <layout-header></layout-header>
            <RouterView :key="$route.fullPath"></RouterView>
            <layout-footer></layout-footer>
        </div>
    </template>
    ```

 2. 监听路有变化，变化之后再执行数据更新操作

    `Category/index.vue`

    ```ts
    import { onBeforeRouteUpdate } from 'vue-router';

    const categoryData = ref({})
    const route = useRoute()
    const getCategory = async(id = route.params.id) => {   //这里设一个默认值，如果参数没有变化，则
        const res = await getTopCategoryAPI(route.params.id)  
        categoryData.value = res.result
    }
    onMounted(() => getCategory())
    //目的：路由参数变化的时候，把分类数据重新发送
    const bannerStore = useBannerStore()
    onBeforeRouteUpdate((to) => {        
        //存在一个问题：使用最新路由参数拿不到参数id（因为这个钩子函数先执行）
        getCategory(to.params.id)
    })
    ```

> 这两种各有千秋：
>
> ​	第一种简单粗暴，销毁重建
>
> ​        第二种节约性能，只改变参数
