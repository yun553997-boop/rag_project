# 表单校验

ElementPlus表单组件内置表单验证[Form 表单 | Element Plus](https://element-plus.org/zh-CN/component/form)：

|    标签名    | 所在组件位置 |          功能          |
| :----------: | :----------: | :--------------------: |
|   el-from    |    最外层    | 绑定表单对象和规则对象 |
| el-from-item |    次外层    |   绑定使用的规则字段   |
|   el-input   |   标签本身   |    双向绑定表单数据    |

校验要求：

> 用户名：不能为空，字段名为 account
> 密码：不能为空且为6-14个字符，字段名为 password
> 同意协议：必选，字段名为 agree

## 1 表单初步校验

### 1.1 配置校验

1. 编写验证规则

   > 规则：
   >
   > 1. required：必填（选）项
   > 2. message：错误提示信息
   > 3. trigger：触发时机（blur：失去光标；change：文本发生变化）

   ```vue
   <script setup>
   import { ref } from 'vue'
   // 表单数据
   const userInfo = ref({
     account: '1311111111', 
     password: '123456',
     agree: true
   })

   // 规则数据对象
   const rules = {
     account: [
       { required: true, message: '用户名不能为空', trigger: 'blur' }
     ],
     password: [
       { required: true, message: '密码不能为空', trigger: 'blur' },
       { min: 6, max: 14, message: '密码长度要求6-14个字符', trigger: 'blur' }
     ],
     agree: []
   }
   </script>
   ```

2. 绑定相应标签

   > el-from:
   >
   > ​     2.绑定数据（:model="userInfo" ）
   >
   > ​     3.表单验证规则（:rules="rules"）
   >
   > ​     4.是否在输入框中显示校验结果反馈图标（status-icon）
   >
   > el-form-item:
   >
   > ​     1.model 的键名（prop）**必填**

   ```vue
   <el-form ref="formRef" :model="userInfo" :rules="rules" label-width="60px" label-position="right"  status-icon>
     <el-form-item  label="账户" prop="account">
       <el-input v-model="userInfo.account"/>
     </el-form-item>
     <el-form-item label="密码" prop="password" >
       <el-input v-model="userInfo.password"/>
     </el-form-item>
     ...
     <el-button size="large" class="subBtn">点击登录</el-button>
   ```

### 1.2 自定义校验

- 为什么需要自定义校验：

  当Element Plus配置的规则不能满足实际开发，就会使用


- 自定义校验规则：

  ```js
  {
      validator(rule, value, callback) => {
          //rule:即为校验规则对象
          //value:当前表单内容
          //calback:校验处理函数，无论通过与否都执行
          if (value === '') {
    		  callback(new Error('提示语'))
    		} else if (value !== ) {
    		  callback(new Error("提示语"))
    		} else {
    		  callback()  //满足条件，通过
    		}
      }
  }
  ```

1.  编写校验规则

   ```ts
   const rules = {
     account: [],
     password: [],
     agree: [
       {
         validator: (rule, val, callback) => {
           return val ? callback() : new Error('请先同意协议')
         }
       }
     ]
   }
   ```

2. 绑定相应标签

   ```vue
   <el-form  ...>
     ...
     <el-form-item label-width="22px" prop="agree">
       <el-checkbox  size="large" v-model="userInfo.agree">
         我已同意隐私条款和服务条款
       </el-checkbox>
     </el-form-item>
   </el-form>
   ```

## 2 表单统一校验

- 为什么需要统一校验：

  因为每个表单域都有**自己**的校验**触发事件**，如果不触发就不会校验，因此需要在用户点击登陆时，进行统一  校验

- 怎么统一校验：

  通过Element Plus内置方法`validata`。他是el-from的实例方法，只要校验，就会对整个表单进行校验

  ```js
  formRef.validate((valid) => {      //它有一个参数valid，当通过所有验证，它就为true
      if (valid) {
          ...                      //通过怎么样
      } esle {
          ...                      //没通过又怎么样
          return false
      }
  })
  ```

1. 标记标签

   ```vue
   <el-form ref="formRef" ...>
     ...
   </el-form>  
   ```

2. 获取from实例做统一验证

   ```ts
   //表单统一校验
   const formRef = ref(null)
   const dologin = () => {
     formRef.value.validator((valid) => {
       console.log(valid)
       //以valid作为判断条件，如果校验成功才执行登录逻辑
       if (valid) {
         //Login
       }
     }) 
   }
   ```

# 实现登录、pinia管理

1. 封装接口

   ```javascript
   import httpInstance from '@/utils/http.js'
   export function loginAPI({ account, password }) {
       return httpInstance ({
           url:'/login',
           method: 'POST',
           data: {
               account,
               password
           }
       })
   }
   ```

2. Pinia管理用户数据

   > 基本思想：Pinia负责用户数据相关的state和action，组件中只负责触发action函数并传递参数

   `stores/user.ts`

   ```ts
   import { defineStore } from "pinia";
   import { loginAPI } from '@/apis/user'
   export const useUserStore = defineStore('user', () => {
       
       const getUser = async({ account, password }) => {
           const res = await loginAPI({ account, password })
       }
       return {
           getUser
       }
   })
   ```

3. 触发登陆跳转

   `Login/index.vue`

   ```ts
   import { ref } from 'vue'
   import { useRouter } from 'vue-router'
   import { ElMessage } from 'element-plus'
   import 'element-plus/theme-chalk/el-message.css'
   import { useUserStore } from '@/stores/user'

   const formRef = ref(null)
   const router = useRouter()
   const userstore = useUserStore()

   // 表单统一校验并完成登录
   const dologin = () => {
     formRef.value.validate(async (valid) => {
       // 以valid作为判断条件，如果校验成功才执行登录逻辑
       if (valid) {
         try {
           await userstore.getUser({
             account: userInfo.value.account,     //将用户输入的数据提交
   ```

# 数据持久化

## 1 pinia用户数据持久化

> 用户数据在Token（用来标识当前用户是否登录）中，而Token持续一段时间会过期。
>
> Pinia的储存是基于**内存（本地）**的，刷新就丢失，为保持登陆状态，需要持久化

官网：[开始 | Pinia Plugin Persistedstate](https://prazdevs.github.io/pinia-plugin-persistedstate/zh/guide/)

1. 安装

   ```bash
   npm i pinia-plugin-persistedstate
   ```

2. 在入口文件进行插件注册

   ```ts
   import { createPinia } from 'pinia'
   import piniaPluginPersistedstate from 'pinia-plugin-persistedstate'

   const pinia = createPinia()
   pinia.use(piniaPluginPersistedstate)
   ```

3. 使用

   `stores/user.ts`

   ```js
   export const useUserStore = defineStore('user', () => {
       ...
   },{
       persist: true,          //持久化
   })
   ```

## 2 请求拦截器携带Token

- 为什么要携带Token：

  很多接口如果想要获取数据必须要带着有效的Token信息才可以

- 为什么在拦截器中：

  拦截器中做一次，用到axios实例的其他都可以拿到



```javascript
// axios请求拦截器
httpInstance.interceptors.request.use(config => {
  // 1. 从pinia获取token数据
  const userStore = useUserStore()
  // 2. 按照后端的要求拼接token数据
  const token = userStore.userInfo.token
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
}, e => Promise.reject(e))
```

# 路由守卫

- 路由守卫概述：

  路由守卫是 Vue Router 提供的一种机制，允许你在路由导航发生前、发生时、发生后执行一些自定义逻辑，主要用于权限控制、数据预加载、页面跟踪等。

- 三种主要的路由守卫类型

- 1. 全局守卫（最常用）：作用于整个应用的所有路由

- 2. 路由独享守卫：只作用于特定路由

- 3. 组件内守卫：在组件内部定义

## 1 全局前置守卫 `beforeEach`

**执行时机**：在路由跳转前执行
**用途**：登录验证、权限检查

```js
router.beforeEach((to, from, next) => {
  // to: 即将要进入的目标路由对象
  // from: 当前导航正要离开的路由对象
  // next: 函数，必须调用这个函数来 resolve 这个钩子
})
```

`Routes/user.ts`

```ts
// 添加路由守卫
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  
  // 定义不需要登录就可以访问的白名单
  const whiteList = ['/login','/',]
  
  // 如果有token，表示已登录
  const token = userStore.userList?.token
  
  if (token) {
    // 已登录状态，如果访问登录页，重定向到首页
    if (to.path === '/login') {
      next('/')
    } else {
      next()
    }
  } else {
    // 未登录状态
    if (whiteList.includes(to.path)) {
      // 在白名单中，直接放行
      next()
    } else {
      // // 不在白名单中，重定向到登录页
      next('/login')
    }
  }
})
```

# 退出登录实现

> 基础思想：
>
> 1. 清除用户信息
> 2. 跳转到登录页

1- 新增清除用户信息action

`stores/user.ts`

```javascript
 // 退出时清除用户信息
  const clearUserInfo = () => {
    userInfo.value = {}
  }
  return {
     clearUserInfo 
  }
```

2- 组件中执行业务逻辑

```vue
<script setup>
import { useUserStore } from '@/stores/userStore'
import { useRouter } from 'vue-router'
const userStore = useUserStore()
const router = useRouter()

// 退出登录业务逻辑实现
const confirm = () => {
  // 1.清除用户信息 触发action
  userStore.clearUserInfo()
  // 2.跳转到登录页
  router.push('/login')
}
</script>

<el-popconfirm @confirm="confirm" title="确认退出吗?" confirm-button-text="确认" cancel-button-text="取消">
  ...
</el-popconfirm>
```
