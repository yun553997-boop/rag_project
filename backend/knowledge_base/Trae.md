# Trae使用教程



## 1 Trae三大核心功能

1. Tab-Cue
2. 智能体聊天模式
3. Editor内置AI编码

### 1.1 Tab-Cue

说明：Tab-Cue是其核心功能，通过AI驱动智能补全代码，和上下文感知能力提高编码效率

配置：设置 -> CUE -> Tab-Cue，开启/关闭

主要功能：

1. 代码补全

   ```
   public class ArraySort{
   	//编写一个方法，可以接收数组，并排序，排序方法使用冒泡排序
   	
   	//编写一个方法，可以接收数组，并打印数组
   	
   	//编写一个main方法，测试sort方法
   	
   }
   ```

   > 在编写提示词的过程中，会自动生成代码，此时按Tab，确认即可

2. 智能代码重写

   ```
   public class ArraySort{
   	//编写一个方法，可以接收数组，并排序，排序方法使用冒泡排序，从小到大排序
   	
   	。。。
   }
   ```

   > 此时边修改，代码也会自动变化，此时按Tab

3. 多行协同优化

   即，多行批量修改，智能体会预测，此时按Tab确认

4. 光标位置预测

5. 接收部分和拒绝

   接受全部：按Tab

   接收部分：按`ctrl`+`<- `  或者 `ctrl`+ `->` 进行选择，跳到目标位置，按`Esc`删除剩余的

   ​



### 1.2 智能体聊天模式

#### 1.2.1 内置智能体

内置5个智能体：

1. Chat

   使用场景：编写代码（不能创建文件）

2. Builder

   适用场景：端到端执行（覆盖前端+后端+数据库+中间件，一条完整链路，中间没有人为干预）常规开发任务

3. Builder with MCP

   使用场景：支持所有配置MCP Services

4. SOLO Builer

   > Trae提供了两种开发模式：IDE模式、SOLO模式
   >
   > IDE模式：人机协作
   >
   > SOLO模式：AI自主

   使用场景：端到端快速开发，适合个人和小团队

   使用步骤：

   1. 创建文件夹
   2. 集成Supabase服务（用于用户认证和数据存储）
   3. 创建项目，并连接至当前Trae项目（点击创建项目，填写项目名称等信息，提交）
   4. 点击连接（连接刚创建的项目）
   5. 写提示词
   6. 调试代码

5. SOLO Coder

   使用场景：复杂项目开发，擅长项目迭代，任务规划、自主编排智能体

   > Plan（计划）模式：先和用户制定计划，再执行

   使用步骤:

   1. 创建文件夹
   2. 写提示词（先给出一个计划文档，进行沟通，直至计划满意），点击执行




#### 1.2.2 自定义智能体

在设置内的智能体自定义设置

步骤：

1. 命名
2. 写提示词
3. 勾选默认开启智能体生成，生成

例如：

1. 命名：MCP自动化助手
2. 写提示词：结合多种MCP工具，帮你完成文件操作、网页搜索、截图等重复性工作
3. 勾选默认开启智能体生成，生成

此时，就可以完成其他操作，不限于生成程序代码，写一些脚本，完成提示词的操作（生成图片、文件分类等）

| 维度     | 脚本 (Script)                                                | 编译型程序 (Compiled Program)                                |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 代表文件 | `.py`, `.js`, `.sh`, `.php`                                  | `.exe` (Windows), `.app` (Mac), 二进制文件                   |
| 代表语言 | Python, JavaScript, PHP, Shell                               | C, C++, Go, Rust                                             |
| 运行方式 | 解释执行：运行时需要一个“翻译官”（解释器）逐行翻译给电脑听。 | 直接执行：运行前已翻译成机器码（0和1），电脑直接听懂，无需翻译官。 |
| 修改难度 | 极易：用记事本就能改，改完保存即生效。                       | 困难：需要修改源代码，重新编译、打包才能生效。               |
| 运行效率 | 相对较慢：因为运行时要边翻译边跑。                           | 极快：直接由 CPU 执行，性能最大化。                          |
| 依赖环境 | 强依赖：电脑里必须先装 Python/Node.js 等环境才能跑。         | 弱依赖：通常自带所有依赖，双击即跑（尤其是 Go/Rust 等）。    |



#### 1.2.3 常用配置

1. Auao Mode

   基于效率与速度选择最优模型（性价比）

2. Max Mode

   适用于复杂任务，支持长上下文（不计成本，对质量要求高，按Token计费）

3. 指定模型

   自选超级模型，例如，GPT-5.2

4. 自定义模型

   需要自己获取模型的API密钥，然后添加（一般自带的就够用了）



### 1.3 Editor内AI编码

说明：直接在编辑器窗口生成新代码，或者编辑现有代码（适用于精修代码）

使用方法：

- 在新文件中使用Ctrl+I，添加提示词
- 选中相应代码，使用Ctrl+I，添加提示词，用于修改或添加新功能



## 2 Trae的其他功能

### 2.1 Trae上下文

上下文（Context）指AI工具回答问题，或者帮助你理解

主要有三大功能;

1. 代码索引管理

   > 说明：对工作区的代码进行全局索引

   使用步骤：

   1. 检查是否构建成功：设置 -> 上下文，构建成功显示100%
   2. 在对话框中输入`#`，选择Workspace
   3. 提出做什么，例如：`Workspace 有哪些文件`

2. 忽略文件

   > 说明：通过`.ignore`文件添加到根目录，来控制那些文件/文件夹被忽略
   >
   > 作用：提升索引速度

   使用步骤：

   1. 点击创建忽略文件：设置 -> 上下文 -> 创建忽略文件

   2. 填写忽略的文件名

      ```ignore
      *.html              //忽略所有html文件
      node_modules/       //忽略node_modules文件夹
      ```

3. 文档集

   > 说明：通过URL、本地上传的方式添加文档集作为上下文与AI的问答

   使用步骤：

   1. 点击上下文 ->  添加文档集 -> 输入URL（或者本地，一般都用官网）
   2. 在对话框，点击`#`，选择添加的文档，填入要求



### 2.2 Rules规则

说明：给AI生成结果添加规则和限制，让AI生成的代码复合团队规范

两个规则：

|          |       项目规则（Project Rules）        |     用户规则（User Rules）     |
| :------: | :------------------------------------: | :----------------------------: |
| 作用范围 | 仅对当前项目生效，团队成员共享相同规则 |    对所有项目生效，个人专属    |
| 使用场景 |  同意团队编码规范（技术栈、依赖版本）  | 个人习惯（通用偏好和工作风格） |

优先级：项目规则 > 用户规则

如何配置：

1. 设置 -> 规则与技能 -> 个人规则/项目规则 -> 创建
2. 直接用中文描述即可



### 2.3 `#`符号作用

在对话框中引用上下文

1. 引用文件

   输入`#` -> 选择`File` -> 输入文件名

2. 引用文件夹

   输入`#` -> 选择`Folder` -> 输入文件夹名

3. 引用代码

   输入`#` -> 选择`Code` -> 输入选择器（CSS）/输入方法名（JS）

4. 引用文档集（见上文）

5. 引用工作区（见上文）

6. 引用Web

   > 引用Web主要是用于联网搜索

   输入`#` -> 选择`Web` 

7. 引用Problems

   > 引用Problems，AI会分析问题标签中所有问题，并提出解决方案

   输入`#` -> 选择`Problems` -> 输入文件名

   ```
   #Problems分析问题，并给出解决方案
   ```

## 3 Trae实战案例

### 3.1 Trae+Figma生成前端页面

Figma简介：用来设计用户界面（UI）和用户体验（UX）的工具

目标：将Figma的UI页面自动生成网页页面

使用：

1. 登陆[Figma](https://www.figma.com) -> 注册

2. 获取token：

   左上角头像 -> settings -> Security（导航条）-> Generate new token -> 填写名字、勾选权限（全勾） -> 创建 -> 复制token

3. 添加Figma MCP:

   设置 -> MCP -> 市场添加 -> 搜索figma并添加 -> 填入token（不用管下面的配置代码）-> 确认

4. 获取（目标）UI组件地址：

   > 这个UI界面在Figma有现成的，也可以自己设计、自己创建（找UI设计师）

   目标页面（右键） -> Copy/Paste as -> Copy link to selection(复制地址)

5. UI页面 ---> 网页页面:

   对话框 -> 选择Builder with MCP -> #地址 

   ```
   #`https://www.figma...` 还原设计页面代码，并帮我生成页面所需要的所有图片
   ```



> 当然，如果使用次数多，可以将其创建为专属（自定义）智能体
